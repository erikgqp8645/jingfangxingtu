# 经方星图 (JingFang Graph) - 数据接入与转化规范

为了将任意一门中医经典（如《伤寒论》、《温病条辨》、《针灸甲乙经》等）完美转化为“星图”格式，我们需要遵循严格的数据文件夹结构和 JSON/TypeScript 数据对象规范。

本指南将指导您如何拆解经典，并将其注入到系统中。

## 1. 标准文件夹结构

建议在后续开发中，将底层数据从单一的 `data.ts` 抽离，采用以下标准的文件夹结构来进行维护。
您可以根据这个规范来整理新的经典书籍资料：

```text
/src/database/
│
├── catalog.ts                  # 全局汇总总目录（控制左侧导航栏的下拉和树状菜单）
│
├── books/                      # 【核心锚点库】存放中医经典原文
│   ├── shanghan_lun/           # 以拼音或英文命名具体的书籍文件夹
│   │   ├── index.ts            # 该书籍的元数据（书名、作者、章节目录）
│   │   └── clauses/            # 存放该书籍下所有条文的独立文件
│   │       ├── clause_263.ts
│   │       ├── clause_264.ts
│   │       └── clause_265.ts   # 具体某一条文的图谱和解析数据
│   │
│   └── jingui_yaolue/          # 《金匮要略》文件夹
│       ├── index.ts
│       └── clauses/
│           └── clause_13.ts
│
└── plugins/                    # 【多维解析图层库】存放跨书籍共用的注解和理论库
    ├── neijing.ts              # 《黄帝内经》底层机制知识库
    ├── nanjing.ts              # 《难经》知识库
    ├── benjing.ts              # 《神农本草经》物性构成知识库
    ├── bingyuan.ts             # 《诸病源候论》病理机制知识库
    └── mingli.ts               # 后世注家（如《伤寒明理论》等）逻辑翻译库
```

## 2. 数据格式说明

为了能被 D3.js 知识图谱引擎和 React 组件正确渲染，每个条文的数据必须严格遵循 `ClauseData` 接口。

### 2.1 核心图谱支持的节点类型 (Node Types)
在提取一段条文时，请将其核心概念抽离为以下 5 种类型的节点，并赋予对应的颜色和图标：
*   `anchor`（锚点，通常为条文本身的编号或主旨）
*   `symptom`（症状，如：头痛、发热、脉弦细）
*   `theory`（理论/病机，如：属少阳、中气不足、木火刑金）
*   `formula`（方剂/治法，如：小柴胡汤、不可发汗）
*   `book`（书籍典故，引用的其他经典来源，如：《黄帝内经》、《诸病源候论》）

### 2.2 核心条文格式 (Clause Template)
请参考同目录下的 `templates/clause_template.ts` 中的详细字段。

### 2.3 多维解析图层格式 (Plugins Format)
右侧面板的“多维解析层”是本系统的核心魅力。当你录入一条经典原文时，你需要去搜索其他典籍中能解释这句原文背后的逻辑，并填入 `plugins` 字段中。

支持的图层类型包括（不限于）：
*   `neijing`: 机制底层（内经/难经）
*   `bingyuan`: 病理机制（诸病源候论/其他病理专著）
*   `mingli`: 逻辑翻译（成无己、尤在泾、徐大椿等后世注家的白话逻辑）
*   `benjing` / `tangye`: 物质与生化（神农本草经、汤液经方、药物性味解析）
