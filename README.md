# 经方星图

这是一个把中医经典条文、关键词和关联解析结果放到同一张关系图里查看的项目。

它不是传统那种很重的“知识图谱”平台，而是一个更直接、更实用的“关系图”工具。

你可以把它理解成一条清晰的链路：

`条文 -> 关键词 -> 关联解析来源片段`

系统会把这条链路展示在页面中，方便查看、补词、排查和持续整理数据。

如果你是第一次打开这个项目，只先记住下面 3 句话就够了：

1. 平时主要改 `data/`
2. 关系是靠 `keywords（关键词）` 建起来的
3. 改完文件后，先刷新页面检查，再决定要不要同步数据库

---

## 这个项目现在能做什么

当前已经支持：

- 查看经典目录和条文
- 查看某条条文对应的关键词
- 在“正文原文区”选中文字并右键添加关键词
- 删除已有关键词
- 根据关键词自动匹配关联解析来源
- 在右侧面板和关系图中显示命中结果
- 用 SQLite（轻量数据库）保存运行时数据
- 保留 JSON（数据文件格式）和 TXT（文本文件）作为日常维护入口

---

## 页面怎么理解

页面大致可以分成三部分：

- 左侧：经典目录、章节、条文列表
- 中间：当前条文正文、关键词、关系图
- 右侧：根据关键词命中的关联来源和来源片段

这个项目最核心的工作，就是让“当前条文”和“它命中的解析内容”之间的关系更容易看清楚。

---

## 数据是怎么工作的

这里有一条最重要的原则：

`data/` 是源数据。  
`SQLite` 是运行时副本。

也就是说，平时维护时，优先修改 `data/` 目录里的文件；数据库主要用于读取、检索和运行，不建议直接手工改数据库。

主流程是：

1. 在 `data/` 里维护经典条文 JSON（数据文件格式）
2. 在 `data/` 里维护关联解析 TXT（文本文件）或 JSON（数据文件格式）
3. 系统读取条文中的 `keywords`（关键词）
4. 用这些关键词去匹配关联解析来源
5. 把命中的结果显示在右侧和关系图里
6. 如有需要，再同步到 SQLite（轻量数据库）

## 目录说明

项目里最重要的目录如下：

```text
jingfangxingtu/
├─ data/                    # 源数据目录
│  ├─ jingdianconfig.json   # 经典目录配置
│  ├─ guanlianjiexiconfig.json
│  ├─ 经典/                  # 条文 JSON
│  └─ 关联解析/              # 解析 TXT / JSON
├─ docs/                    # 文档目录
├─ scripts/                 # 数据处理和数据库脚本
├─ server/                  # 本地接口
├─ shared/                  # 共用逻辑
├─ src/                     # 前端页面代码
└─ storage/
   └─ app.db               # SQLite 数据库文件
```

## 快速开始

如果你只是想先把项目跑起来，请照下面 4 步做：

1. 安装依赖
2. 启动项目
3. 打开页面
4. 随便点开一条条文，看左中右三栏是否正常显示

安装依赖：

```bash
npm install
```

启动开发环境：

```bash
npm run dev
```

然后打开：

[http://localhost:3000](http://localhost:3000)

打开后，建议马上检查这 4 件事：

1. 左侧是否能看到经典目录
2. 中间是否能看到条文正文
3. 关键词区域是否有词
4. 右侧是否能看到关联命中或命中数量

## 最常见的 4 类工作

如果你后面准备长期维护，这个项目最常做的事情基本只有下面 4 类：

1. 给已有条文添加或删除 `keywords（关键词）`
2. 给 `data/关联解析/` 新增 JSON（数据文件格式）或 TXT（文本文件）
3. 给 `data/经典/` 新增条文 JSON（数据文件格式）
4. 在需要的时候把文件数据同步到 SQLite（轻量数据库）

也就是说，这个项目虽然看起来目录很多，但真正经常动手的地方并不多。

---

## 不要先做的事

为了避免新手一上来就走弯路，这里明确写 4 条“不推荐先做”的事：

1. 不要先去改 `storage/app.db`
2. 不要先去改 `dist/` 里的文件
3. 不要一口气批量改很多文件再一起检查
4. 不要把特别泛的词直接当 `keywords（关键词）`
