# 经方星图

这是一个把中医经典条文、关键词和关联解析结果放到同一张关系图里查看的项目。

它现在不是复杂的“知识图谱”平台，而是一个更实用的“关系图”工具：

- 左侧看经典目录和条文
- 中间看当前条文和关系图
- 右侧看根据关键词命中的解析来源

这个项目的核心目标很简单：

把“条文 -> 关键词 -> 解析来源片段”这条链路清楚地展示出来，方便学习、整理和持续补数据。

---

## 这个项目能做什么

当前已经支持：

- 查看经典条文
- 查看条文关键词
- 手动添加关键词
- 删除关键词
- 根据关键词自动匹配关联解析
- 用关系图展示条文、关键词和来源片段之间的关系
- 用 SQLite 保存运行时数据
- 保留 JSON 和 TXT 作为最直接的数据维护来源

---

## 项目现在的工作方式

这个项目的主要数据流是：

1. 先在 `data/` 目录维护经典条文 JSON 或关联解析 TXT / JSON
2. 前端读取条文内容和关键词
3. 系统根据关键词去匹配关联解析来源
4. 把命中的结果显示到右侧面板和关系图中
5. 如果需要，可以把源数据同步到 SQLite 数据库

这里最重要的一条原则是：

`data/` 是源数据。  
SQLite 是运行时副本。

也就是说，平时维护内容时，优先改 `data/`，不要把数据库当成手工编辑入口。

## 目录说明

项目里最重要的目录是这些：

```text
jingfangxingtu/
├─ data/                    # 源数据目录
│  ├─ jingdianconfig.json   # 经典目录配置
│  ├─ guanlianjiexiconfig.json
│  ├─ 经典/                  # 条文 JSON
│  └─ 关联解析/              # 解析 TXT / JSON
├─ docs/                    # 文档目录
├─ scripts/                 # 数据处理和数据库脚本
├─ src/                     # 前端代码
├─ server/                  # 本地接口
├─ shared/                  # 共用逻辑
└─ storage/
   └─ app.db               # SQLite 数据库
```

## 新手最该先知道的事

如果你是第一次接手这个项目，只要先记住下面几条：

1. 平时优先修改 `data/` 里的文件
2. `keywords` 是关键词数组
3. 关联关系主要来自“关键词命中”
4. 数据改完后，如有需要，再同步到 SQLite
5. 看不懂代码时，先看 `docs/`
