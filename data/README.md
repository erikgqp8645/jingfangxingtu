# data 目录说明

## 术语对照

为了让新手更容易看懂，下文常见英文词按下面理解：

- `data（数据目录）`
- `JSON（数据文件格式）`
- `TXT（文本文件）`
- `config（配置）`

`data/` 是这个项目最重要的数据目录。

如果你是新手，请优先维护这里的文件。

原因很简单：

- 最直观
- 最容易看懂
- 最容易备份
- 出问题时最好排查

---

## 1. 目录结构

```text
data/
├─ jingdianconfig.json
├─ guanlianjiexiconfig.json
├─ 经典/
├─ 关联解析/
└─ 白话解析/
```

---

## 2. 每个文件夹是做什么的

### `jingdianconfig.json`

经典目录配置文件。

控制左侧有哪些书、哪些章节、哪些条文。

### `guanlianjiexiconfig.json`

关联解析来源配置文件。

控制系统会去哪些来源里搜索关键词。

### `经典/`

放经典条文 JSON（数据文件格式）。

一条条文通常对应一个 JSON（数据文件格式）文件。

### `关联解析/`

放关联解析资料。

支持两种格式：

- JSON（数据文件格式）
- TXT（文本文件）

### `白话解析/`

预留目录。

如果以后要把白话解释单独拆出来，可以放这里。

---

## 3. 新手最重要的原则

1. 平时先改 `data/`
2. 改完先刷新页面检查
3. 没问题再考虑导数据库
