# 经方星图 (JingFang Graph) - 数据接入与转化规范

为了将任意一门中医经典（如《伤寒论》、《温病条辨》、《针灸甲乙经》等）完美转化为“星图”格式，我们将使用 JSON 格式作为纯数据存储，方便非开发人员直接管理和编辑。

本指南将指导您如何拆解经典，并将其通过 JSON 配置注入到系统中。

## 1. 标准文件夹结构

为了实现核心数据的解耦，我们在 `/data` 目录下建立了三个核心数据维度的文件夹：

```text
/data/
│
├── 经典/                       # 【节点与网络】：存放中医著作的本体结构与图谱骨架
│   ├── catalog.json            # 全局导航树状目录
│   └── 伤寒论/
│       └── 条文/
│           ├── 265.json        # 仅包含 nodes, links 等图谱骨架数据
│
├── 多维解析/                   # 【知识图层】：存放不同理论视角对该条文的深度解析
│   └── 伤寒论/
│       └── 265/
│           ├── 内经机制.json     # 对应原体系的 plugins.neijing
│           ├── 诸病源候.json     # 对应原体系的 plugins.bingyuan
│           ├── 后世各家注解.json  # 对应原体系的 plugins.mingli
│           └── 汤液经法五味.json
│
└── 白话解析/                   # 【翻译说明】：存放面向现代读者的白话文翻译
    └── 伤寒论/
        ├── 265.json            # 包含该条文的现代白话翻译和基础解释
```

## 2. 数据格式说明 (JSON)

JSON 是严格的数据格式（不支持 `//` 等注释），请直接参考 `templates` 文件夹下的规范示例。

### 2.1 核心图谱支持的节点类型 (Node Types)
在提取一段条文时，请将其核心概念抽离为以下 5 种类型的节点，并在 JSON 的 `nodes` 数组中赋予对应的 `type`：
*   `anchor`（锚点，通常为条文本身的编号或主旨）
*   `symptom`（症状，如：头痛、发热、脉弦细）
*   `theory`（理论/病机，如：属少阳、中气不足、木火刑金）
*   `formula`（方剂/治法，如：小柴胡汤、不可发汗）
*   `book`（书籍典故，引用的其他经典来源，如：《黄帝内经》、《诸病源候论》）

### 2.2 核心条文与多维解析格式 (Clause JSON)
请参考 `data_standard/templates/clause_template.json`。
右侧面板的“多维解析层”配置在 `plugins` 字段中。支持的图层键名（Key）包括：
*   `neijing`: 机制底层（内经/难经）
*   `bingyuan`: 病理机制（诸病源候论/其他病理专著）
*   `mingli`: 逻辑翻译（成无己、尤在泾、徐大椿等后世注家的白话逻辑）
*   `benjing` / `tangye`: 物质与生化（神农本草经、汤液经方、药物性味解析）
