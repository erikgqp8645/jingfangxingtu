# 关联解析 (Relation Sources)

该文件夹用于存放“关系检索来源”。

系统不会再把这里的数据理解为“多维解析插件层”，而是把它视为一组可被关键词检索的来源文本：

- 若存在同名 `.json`，优先按结构化 JSON 检索
- 若不存在可用 `.json`，则回退到 `.txt` 检索

## 当前职责

当某条经典条文包含关键词时，系统会到本目录下登记过的数据源中查找同名关键词，并返回命中的篇名或摘要片段。

因此，这里的文件职责是：

- 提供可被关键词命中的结构化内容
- 提供可被全文检索的原始文本
- 为右侧关联面板和中部关系图提供来源节点

## 文件命名

文件名应与 `data/guanlianjiexiconfig.json` 中的 `fileBaseName` 一致。

例如：

- `437-黄帝内经素问.json`
- `437-黄帝内经素问.txt`
- `467-伤寒寻源.txt`

## JSON 推荐结构

推荐按“关键词索引”组织：

```json
{
  "谵语": [
    {
      "title": "热病论",
      "content": "……"
    }
  ]
}
```

每个命中项至少应包含：

- `title`
- `content`

## TXT 使用建议

TXT 文件建议保留可切分的标题、篇名或段落标记，便于系统：

- 按篇名或标题切分
- 命中关键词后返回局部摘要

如果某来源已经提供高质量 JSON，应优先维护 JSON，而不是继续依赖 TXT 模糊匹配。
