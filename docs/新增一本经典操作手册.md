# 新增一本经典操作手册

这份文档只讲一件事：

`怎么把一本新的经典加入项目。`

如果你是第一次做，请严格按顺序操作。

不要跳步骤。

---

## 1. 先理解目标

把一本新的经典加进项目，最少要完成 4 件事：

1. 有经典条文文件
2. 有经典目录配置
3. 页面里能看到这本书
4. 条文点开后有正文、有白话、有关键词

数据库不是第一步。

数据库是最后按需要再做。

---

## 2. 最推荐的顺序

请按这个顺序做：

1. 先准备这本经典的条文 JSON
2. 再修改 `data/jingdianconfig.json`
3. 刷新页面检查
4. 需要数据库时再写导入脚本或执行导入

这是最稳的顺序。

---

## 3. 第一步：先想好这本书的编号

每本书要有一个稳定的 `id`。

例如：

- `shanghan`
- `jingui`
- `wenbing`

假设你现在新增一本《脾胃论》，我们举例用：

```text
piweilun
```

这个 `id` 后面会反复用到。

---

## 4. 第二步：创建经典目录

先在：

[data/经典](C:\Users\hxst01\Documents\aicoding\jingfangxingtu\data\经典)

下面新建一个文件夹。

例如：

```text
data/经典/脾胃论/
```

## 5. 第三步：创建条文 JSON 文件

假设这本书第一章第一条的编号是：

```text
piweilun-01-01
```

那你可以先建文件：

[piweilun-01-01.json](C:\Users\hxst01\Documents\aicoding\jingfangxingtu\data\经典\脾胃论\piweilun-01-01.json)

内容示例：

```json
{
  "id": "piweilun-01-01",
  "title": "脾胃论 第一章 第一条",
  "content": "这里写原文。",
  "translation": "这里写白话解释。",
  "keywords": ["脾胃", "中气", "饮食"]
}
```

如果还有第二条，就再建：

[piweilun-01-02.json](C:\Users\hxst01\Documents\aicoding\jingfangxingtu\data\经典\脾胃论\piweilun-01-02.json)

内容示例：

```json
{
  "id": "piweilun-01-02",
  "title": "脾胃论 第一章 第二条",
  "content": "这里写第二条原文。",
  "translation": "这里写第二条白话解释。",
  "keywords": ["脾虚", "胃气", "劳倦"]
}
```

---

## 6. 条文 JSON 必须有哪些字段

最少必须有：

```json
{
  "id": "唯一编号",
  "title": "条文标题",
  "content": "原文",
  "translation": "白话解释",
  "keywords": ["关键词1", "关键词2"]
}
```

### 字段说明

#### `id`

条文唯一编号。

不能重复。

#### `title`

页面上显示的条文标题。

#### `content`

条文原文。

#### `translation`

白话解释。

可以先留空字符串，但最好逐步补齐。

#### `keywords`

关键词数组。

系统会拿它去匹配关联解析。

## 7. 第四步：修改 `jingdianconfig.json`

打开：

[jingdianconfig.json](C:\Users\hxst01\Documents\aicoding\jingfangxingtu\data\jingdianconfig.json)

在最外层数组里新增一本书。

示例：

```json
{
  "id": "piweilun",
  "name": "《脾胃论》",
  "chapters": [
    {
      "title": "第一章",
      "clauses": [
        {
          "id": "piweilun-01-01",
          "title": "第1条 · 这里写第一条标题",
          "dataFile": "/data/经典/脾胃论/piweilun-01-01.json"
        },
        {
          "id": "piweilun-01-02",
          "title": "第2条 · 这里写第二条标题",
          "dataFile": "/data/经典/脾胃论/piweilun-01-02.json"
        }
      ]
    }
  ]
}
```

---

## 8. `jingdianconfig.json` 的层级怎么理解

一共三层：

1. 书
2. 章节
3. 条文

你可以把它理解成目录树。

---

## 9. 第五步：刷新页面检查

保存之后，刷新：

[http://localhost:3000](http://localhost:3000)

检查下面几件事：

1. 左侧有没有出现《脾胃论》
2. 点击后能不能看到章节
3. 点击条文后中间有没有显示正文
4. 关键词有没有显示
5. 白话解释有没有显示

如果这一步就有问题，先不要导数据库。

先把 JSON 和配置修好。

---

## 10. 常见错误

### 错误 1：`dataFile` 路径写错

例如文件明明在：

```text
data/经典/脾胃论/piweilun-01-01.json
```

却写成了别的目录。

结果：

左侧有条文，点进去内容空白。

### 错误 2：`id` 不一致

例如配置里写：

```json
"id": "piweilun-01-01"
```

但 JSON 里面写成：

```json
"id": "piweilun-1-1"
```

结果：

容易造成后续匹配混乱。

### 错误 3：JSON 格式错误

最常见的是：

- 少逗号
- 多逗号
- 少引号

---

## 11. 什么时候导入数据库

只有在你确认页面文件模式已经没问题时，再考虑数据库。

如果这本书已经稳定，可以做下一步。

### 已有导入脚本时

直接执行对应命令。

### 没有导入脚本时

可以先只用 JSON 跑。

等数据结构稳定后，再补导入脚本。

## 12. 一个完整小例子

假设你新增《脾胃论》。

### 第一步：创建文件

文件：

[piweilun-01-01.json](C:\Users\hxst01\Documents\aicoding\jingfangxingtu\data\经典\脾胃论\piweilun-01-01.json)

内容：

```json
{
  "id": "piweilun-01-01",
  "title": "脾胃论 第一章 第一条",
  "content": "饮食自倍，肠胃乃伤。",
  "translation": "如果吃得过多过杂，就会损伤肠胃。",
  "keywords": ["饮食", "肠胃", "伤"]
}
```

### 第二步：登记到目录

在 `jingdianconfig.json` 里加：

```json
{
  "id": "piweilun",
  "name": "《脾胃论》",
  "chapters": [
    {
      "title": "卷一",
      "clauses": [
        {
          "id": "piweilun-01-01",
          "title": "第1条 · 饮食自倍，肠胃乃伤",
          "dataFile": "/data/经典/脾胃论/piweilun-01-01.json"
        }
      ]
    }
  ]
}
```

### 第三步：刷新页面

如果左侧能看到《脾胃论》，并且条文能正常打开，说明这一步成功了。

---

## 13. 最后记住一句话

新增一本经典时，最稳的顺序永远是：

`先建条文 JSON，再配目录，再检查页面，最后才考虑数据库。`
