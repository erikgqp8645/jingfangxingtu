# 关键词维护操作手册

## 术语对照

为了让不懂英文的用户也能看懂，下文常见英文词按下面理解：

- `JSON（数据文件格式）`
- `TXT（文本文件）`
- `id（编号）`
- `content（正文）`
- `translation（白话解释）`
- `keywords（关键词）`

这份文档只讲一件事：

`怎么添加关键词、删除关键词、检查为什么没有命中。`

如果你是新手，这份文档会比总手册更直接。

如果你只想先记住一句话，请记住：

`先保证关键词真的加进条文里，再去看右侧为什么命中或没命中。`

如果你今天只做一件事，那就做这个最小练习：

1. 找一条条文
2. 右键加一个关键词
3. 看关键词列表有没有出现
4. 看右侧命中有没有变化

这就是关键词维护最核心的一整套闭环。

---

## 1. 先理解关键词是什么

在这个项目里，关键词不是装饰，也不是备注。

关键词是整张关系图的核心开关。

你可以把它理解成：

`条文拿着关键词出去找资料。`

只要条文里有关键词，系统就会用这些关键词去 `data/关联解析/` 里查。

查到了，就会显示到：

- 右侧关联命中面板
- 中间关系图

所以：

`没有关键词，就很难有关系。`

---

## 2. 最常见的三种操作

你平时最常做的，一般就是下面三件事：

1. 添加关键词
2. 删除关键词
3. 排查为什么没有命中

---

## 3. 如何添加关键词

现在项目里有两种添加方法。

### 方法一：页面里右键添加

这是最推荐的方法。

### 操作步骤

1. 打开页面
2. 在左侧找到一本经典
3. 再选中一条条文
4. 在中间“正文原文区”选中一段文字
5. 点击鼠标右键
6. 点击“添加为关键词”

### 例子

正文是：

看这个例子时，只抓一个重点：

`选中的这几个词，都会作为单独关键词写进去。`

```text
少阳不可发汗，发汗则谵语，此属胃，胃和则愈；胃不和，烦而悸。
```

你可以选中：

- 少阳
- 谵语
- 胃不和
- 烦而悸

然后右键添加。

### 这个方法的好处

1. 不需要手改 JSON（数据文件格式）
2. 选中的文字会原样写入
3. 更不容易写错
4. 会自动同步到 JSON（数据文件格式）和数据库

最适合的使用场景是：

- 你正在读某条条文
- 你临时发现一个很合适的词
- 你想马上验证它会不会命中右侧

## 4. 如何用 JSON（数据文件格式）手工添加关键词

如果你不想在页面里点，也可以直接改条文 JSON（数据文件格式）。

如果你是完全不会改 JSON（数据文件格式）的新手，优先还是用页面右键方式。

例如打开：

[265.json](C:\Users\hxst01\Documents\aicoding\jingfangxingtu\data\经典\伤寒论\265.json)

原来可能是：

```json
{
  "id": "265",
  "title": "伤寒论 第265条",
  "content": "伤寒，脉弦细，头痛发热者，属少阳。少阳不可发汗，发汗则谵语，此属胃，胃和则愈；胃不和，烦而悸。",
  "translation": "这里写白话解释。",
  "keywords": ["脉弦细", "头痛发热", "少阳", "谵语"]
}
```

如果你要补两个关键词：

- 胃不和
- 烦而悸

那就改成：

```json
{
  "id": "265",
  "title": "伤寒论 第265条",
  "content": "伤寒，脉弦细，头痛发热者，属少阳。少阳不可发汗，发汗则谵语，此属胃，胃和则愈；胃不和，烦而悸。",
  "translation": "这里写白话解释。",
  "keywords": ["脉弦细", "头痛发热", "少阳", "谵语", "胃不和", "烦而悸"]
}
```

保存后刷新页面就能看到。

看这个示例时，请重点注意：

- `keywords（关键词）` 是一个方括号数组
- 每个关键词都要用引号包起来
- 词和词之间要用逗号隔开

## 5. 如何删除关键词

现在项目已经支持在页面里直接删除。

### 操作步骤

1. 打开一条条文
2. 在关键词列表里找到你要删的词
3. 点击这个词右边的小“删”按钮
4. 在弹出的确认框里点“确定”

### 删除后会发生什么

删除后系统会同时做这些事：

1. 从当前条文的关键词列表里删掉这个词
2. 更新对应条文 JSON
3. 更新数据库里的条文关键词关系
4. 如果这个关键词当前被勾选，它会自动退出筛选
5. 右侧关联命中会重新计算
6. 中间关系图也会重新计算

正常情况下，删除完成后你应该还停留在当前这条条文，不应该自动跳到第一条。

### 特别注意

删除的只是：

`当前条文 -> 这个关键词`

不是删除全项目所有地方的这个关键词。

举例：

- 条文 A 有 `谵语`
- 条文 B 也有 `谵语`

你在条文 A 里删掉 `谵语`，不会影响条文 B。

## 6. 关键词应该怎么选

这是很重要的一部分。

不是所有词都适合当关键词。

### 推荐的关键词

推荐选：

- 表达明确概念的词
- 病机词
- 症状词
- 脉象词
- 证候词

例如：

- 谵语
- 恶寒
- 少阳
- 脉弦细
- 头痛发热
- 胃不和

### 不推荐的关键词

不推荐选太泛的词：

- 病
- 人
- 有
- 热
- 寒

因为这类词命中会太多，关系图会很乱。

## 7. 为什么我加了关键词，右侧却没有命中

这是最常见的问题。

一般按下面顺序排查。

### 第一步：确认这个关键词真的存在于当前条文里

先看关键词列表里是不是已经能看到这个词。

如果列表里都没有，那说明添加还没成功。

### 第二步：确认这个关键词现在是勾选状态

如果你把某个关键词取消勾选了，它就不会参与当前匹配。

所以要看：

这个词有没有勾选。

### 第三步：确认右侧“来源片段”开关是不是打开的

如果你把“来源片段”关掉了，右侧会不显示。

另外还要注意：

如果你只勾选了某几个关键词，那么右侧显示的命中数量也应该跟着变。

### 第四步：确认关联解析里真的有同名词

例如你加的是：

```text
恶寒
```

那就去关联解析里查：

- 有没有 `恶寒`
- 有没有写成别的形式，比如 `恶寒者`

如果写法不同，可能就匹配不到。

### 第五步：确认这个来源有没有登记

就算文件里有内容，如果没有登记到：

[guanlianjiexiconfig.json](C:\Users\hxst01\Documents\aicoding\jingfangxingtu\data\guanlianjiexiconfig.json)

系统也不会去查它。

### 第六步：确认 JSON / TXT 规则

现在系统规则是：

1. 某个关键词先查 JSON
2. JSON 没命中，再回退到 TXT

所以如果某个来源同时有 JSON 和 TXT，不要以为它只会查 JSON。

它是“关键词级别回退”。

这句话可以翻成更容易懂的话：

`同一本来源里，不是整本书只用 JSON 或只用 TXT，而是每个关键词单独决定先查谁、再回退谁。`

## 8. 典型排查例子

目标：

为什么 `恶寒` 没出现在右侧？

### 条文

```json
{
  "id": "1",
  "title": "伤寒论 第1条",
  "content": "太阳之为病，脉浮，头项强痛而恶寒。",
  "translation": "太阳病的特点，是脉浮、头项强痛，而且怕冷。",
  "keywords": ["太阳", "脉浮", "头项强痛", "恶寒"]
}
```

### 排查顺序

1. 看页面里 `恶寒` 有没有出现在关键词列表
2. 看 `恶寒` 有没有被勾选
3. 看右侧来源片段开关有没有打开
4. 看 [437-黄帝内经素问.json](C:\Users\hxst01\Documents\aicoding\jingfangxingtu\data\关联解析\437-黄帝内经素问.json) 里有没有 `恶寒`
5. 如果 JSON 没有，再看 [437-黄帝内经素问.txt](C:\Users\hxst01\Documents\aicoding\jingfangxingtu\data\关联解析\437-黄帝内经素问.txt) 里有没有 `恶寒`
6. 再看来源有没有登记到 `guanlianjiexiconfig.json`

这样一层一层查，基本都能查到原因。

## 9. 如果关键词太多，关系图太乱怎么办

这个时候不要急着删数据。

先做这几件事：

1. 取消勾选一些关键词
2. 先只保留最有价值的关键词
3. 避免使用特别泛的词

比如：

不推荐同时塞进很多泛词：

- 病
- 热
- 寒
- 虚

更推荐保留更具体的词：

- 谵语
- 脉弦细
- 胃不和
- 头痛发热

## 10. 什么时候用页面添加，什么时候用 JSON 添加

### 适合页面添加的时候

- 你正在阅读某条条文
- 你一眼就看到了适合当关键词的词
- 你想快速加一个词试试效果

### 适合 JSON 添加的时候

- 你要一次性批量整理很多关键词
- 你正在做数据清洗
- 你希望统一规范关键词顺序

如果你只是试一个词，优先用页面方式。

---

## 11. 新手最稳的工作法

推荐这样做：

1. 先在页面里添加关键词
2. 立刻看右侧有没有命中
3. 如果效果好，就保留
4. 如果命中太乱，再删掉或取消勾选
5. 如果完全没有命中，再去补关联解析资料

这个方法最适合新手。

因为：

- 看得见
- 反馈快
- 不容易改错很多文件

---

## 12. 最后记住一句话

维护关键词时，最稳的顺序永远是：

`先添加关键词，再看命中，再决定要不要补关联解析或删除关键词。`
