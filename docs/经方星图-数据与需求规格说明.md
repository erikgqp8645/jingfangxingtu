# 经方关系图 — 数据与需求规格说明

**文档来源**：`data/README.md`、`data/关联解析/README.md`、`data/白话解析/README.md`，以及 `data/jingdianconfig.json`、`data/guanlianjiexiconfig.json` 中的需求性说明（草稿）。  
**说明**：规范中引用的 `data_standard/templates/*.json` 在仓库中可能尚未落地，本文档以仓库根目录 **`data/`** 内已写明的内容为准；若后续增补模板，应与此文一并维护。

---

## 1. 目标与范围

### 1.1 业务目标

- 将任意一门中医经典（如《伤寒论》《温病条辨》《针灸甲乙经》等）转化为**关键词关系图**可用的纯数据，以 **JSON** 为主存储，使**非开发人员**可直接编辑与管理。
- 应用侧通过 JSON 配置将数据注入系统，实现**经典目录**、**条文详情**、**关键词关系**与**关联解析来源**的解耦。

### 1.2 范围边界

- **在册需求**：数据目录约定、条文 JSON 结构、关键词关系语义、关联解析加载规则、经典选择与检索联动（见下文）。
- **不在本文展开**：前端具体组件行为（以代码实现为准）。

---

## 2. 标准数据分层（目录结构需求）

为实现核心数据解耦，站点根路径 **`/data`**（本仓库对应**根目录 `data/`**，由 Vite 映射到该 URL）下应建立**三个核心维度**（规范中的目标结构如下；若当前仓库目录尚未完全按此落地，以迁移到此结构为方向）。

```text
/data/
├── jingdianconfig.json         # 经典目录与条文入口配置
├── guanlianjiexiconfig.json    # 关联解析来源配置
├── 经典/                       # 条文详情数据
│   └── 伤寒论/
│       ├── 265.json
│       └── 29.json
├── 关联解析/                   # 关联解析来源，优先 JSON，其次 TXT
└── 白话解析/                   # 可选拆分目录
```

**需求要点**：

| 维度 | 职责 |
|------|------|
| 经典 | 条文详情与关键词 |
| 关联解析 | 按关键词命中的关联文献来源 |
| 白话解析 | 「译」与通俗释义，降低入门门槛 |

## 3. 关系数据需求

### 3.1 条文 JSON

条文级 JSON 只需维护最小字段：

- `id`
- `title`
- `content`
- `translation`
- `keywords`

其中 `keywords` 为建立关系的唯一直接依据。

### 3.2 关系语义

系统中的关系定义为：

- 条文中的某个关键词
- 在某个关联解析来源中命中同名内容
- 即建立文本关联

关系的语义不是病机推演，而是“共享关键词导致的文本关联”。

---

## 4. 关联解析（`关联解析/`）需求

### 4.1 职责

- 存放被检索的关联解析文本与结构化内容。
- 浏览某条条文时，系统根据其 `keywords` 到此目录中的数据源查找同名关键词。
- 可持续新增新的关联来源，**无需修改条文本体结构**。

### 4.2 关联解析加载规则（来自 `guanlianjiexiconfig` 草稿说明）

- 每个数据源通过 `guanlianjiexiconfig.json` 进行注册。
- 每个来源可指向仓库 `data/关联解析/`（运行时 URL：`/data/关联解析/`）下的资源：
  - **若存在**与约定命名一致的 **`.json`**（示例：`437-黄帝内经素问.json`），则 **优先加载 JSON**；
  - **若无 JSON**，则加载 **`.txt`**；在 TXT 模式下，需在文本中**按需要检索的关键词**定位内容（与条文侧给出的检索词配合，如草稿中「从关联解析文本中搜索」）。

---

## 5. 白话解析（`白话解析/`）需求

### 5.1 职责

- 存放经文对应的**现代白话翻译、通俗释义及背景铺垫**。

### 5.2 应用场景

- 面向初学者的现代语境转述。
- 填充详情视图中的**「译」**相关展示，提高可读性与亲和力。

---

## 6. 经典选择与条文配置需求

- **「选择经典」** 的内容在 `jingdianconfig.json` 中定义；结构需能驱动界面中的**经典下拉及下级内容**。
- 每条条文配置通过 `dataFile` 指向 `data/经典/` 下的条文 JSON。
- 条文中的**关键词**（如「谵语、吐逆」）用于从关联解析文本中检索/匹配结果。
- 多书支持：如 **金匮要略、温病条辨** 等，应能**自动出现在经典选择列表**中（与配置结构一致的前提下）。

---

## 7. 数据格式约束

- **JSON 为严格数据格式**：不支持 `//` 等注释；编辑时需保证可被标准解析器解析。
- 具体字段样例以 `data/经典/*.json` 与 `jingdianconfig.json` 的现状为准。

---

## 8. 非功能性需求（摘自规范）

- **可编辑性**：优先让非开发人员通过文件与目录管理数据。
- **可扩展性**：关联解析与白话层可独立增删，不强制改动条文结构。
- **一致性**：数据仅维护仓库根 **`data/`** 一处；构建时复制到 `dist/data/`，无第二套编辑源。
