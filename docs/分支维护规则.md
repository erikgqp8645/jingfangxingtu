# 分支维护规则

## 这份文档是做什么的

这份文档专门规定：

1. `main` 分支怎么维护
2. `codex/windows-desktop-app` 分支怎么维护
3. 哪些内容可以互相同步
4. 哪些内容绝对不能反向带回去

这份规则从现在开始长期生效。

---

## 一句话先记住

`main` 是网页主线。  
`codex/windows-desktop-app` 是桌面版独立分支。  
以后只允许 `main -> codex/windows-desktop-app`，不允许 `codex/windows-desktop-app -> main`。

---

## 两个分支各自负责什么

### 1. `main`

`main` 负责项目的主线内容：

- 经典数据结构
- 关系图功能
- 关键词功能
- 解析来源功能
- 数据维护能力
- 数据库同步能力
- 通用前后端功能

简单理解：

`main` 里放“网页版和桌面版都需要的通用能力”。

### 2. `codex/windows-desktop-app`

`codex/windows-desktop-app` 负责桌面版专属内容：

- Electron（桌面壳）
- Windows 打包
- 安装包
- 图标
- 桌面版启动流程
- 安装后数据目录
- 桌面版发布文档
- 桌面版专属修复

简单理解：

`codex/windows-desktop-app` 里放“只有桌面版才需要的东西”。

---

## 允许同步的方向

### 允许

允许把这些内容从 `main` 同步到 `codex/windows-desktop-app`：

- 通用前端页面改动
- 通用后端接口改动
- 通用数据结构改动
- 通用数据库改动
- 通用脚本文档改动
- 通用 bug 修复

也就是：

只要这项改动不是“桌面版专属”，就可以从 `main` 往桌面版分支同步。

### 不允许

不允许把这些内容从 `codex/windows-desktop-app` 反向同步到 `main`：

- `electron/` 目录改动
- `assets/desktop/` 目录改动
- `build:desktop` 相关逻辑
- Windows 安装包逻辑
- 桌面版启动端口逻辑
- 桌面版发布说明
- 桌面版专属修复

也就是：

`codex/windows-desktop-app -> main` 默认禁止。  
除非先人工确认“这次改动完全是通用能力”，否则不能往 `main` 合并。

---

## 以后实际操作时怎么做

### 情况 A：做通用功能

例如：

- 关系图改进
- 关键词功能改进
- 同步按钮改进
- 数据库接口改进

正确顺序：

1. 先在 `main` 或者从 `main` 切出的功能分支开发
2. 测试通过后合并回 `main`
3. 再把 `main` 合并到 `codex/windows-desktop-app`

记住：

先主线，后桌面版。

### 情况 B：做桌面版专属功能

例如：

- 改 Electron 窗口
- 改安装包名称
- 改桌面图标
- 改打包脚本
- 改桌面版安装后数据目录

正确顺序：

1. 只在 `codex/windows-desktop-app` 开发
2. 只推送到 `codex/windows-desktop-app`
3. 不合并到 `main`

记住：

桌面版专属内容只能留在桌面版分支。

---

## 合并时必须做的检查

每次准备合并前，先问自己下面 4 个问题：

1. 这次改动是不是网页版也需要？
2. 这次改动里有没有 Electron（桌面壳）相关文件？
3. 这次改动里有没有 `assets/desktop/` 相关文件？
4. 这次改动里有没有 `build:desktop`、安装包、图标、Windows 路径相关内容？

如果第 2、3、4 个问题里有一个答案是“有”，那就默认不能进 `main`。

---

## 禁止误操作的硬规则

以后默认遵守下面这几条：

1. 不从 `codex/windows-desktop-app` 直接合并回 `main`
2. 不把桌面版发布文档带回 `main`
3. 不把桌面版图标资源带回 `main`
4. 不把 Electron（桌面壳）专属修复带回 `main`
5. 如果不确定一项改动是否属于桌面版专属，就先判定为“不能进 `main`”

---

## 推荐工作流

推荐长期使用下面这条流程：

1. 通用功能先做在 `main`
2. `main` 稳定后，再同步到 `codex/windows-desktop-app`
3. 桌面版专属功能只做在 `codex/windows-desktop-app`
4. 发布网页主线时看 `main`
5. 发布桌面版时看 `codex/windows-desktop-app`

---

## 当前项目的正式约定

从这份文档开始，当前项目正式采用下面这条约定：

`main` 和 `codex/windows-desktop-app` 同时维护，但职责不同。  
`main` 是主线。  
`codex/windows-desktop-app` 是桌面版独立维护分支。  
以后只做 `main -> codex/windows-desktop-app` 的同步，不做 `codex/windows-desktop-app -> main` 的反向同步。
