# 关键词标准表

## 术语对照

为了让新手更容易看懂，下文常见英文词按下面理解：

- `keywords（关键词）`
- `JSON（数据文件格式）`

这份文档是第二阶段的数据维护标准。

目标很简单：

- 让不同人补关键词时，写法尽量一致。
- 让右侧关联命中更稳定，不要同一个意思写出很多版本。
- 让后面从 JSON 迁移到数据库时，数据更干净。

## 一、先记住一句话

关键词不是“越多越好”，而是“越准越好”。

一个好关键词，应该满足下面 3 个条件：

- 能直接从条文原文里找到。
- 语义比较明确，不是特别空泛的大词。
- 以后在关联解析里也有机会命中。

## 二、优先添加哪些关键词

建议按下面顺序添加。

### 1. 症状词

最优先。

示例：

- 恶寒
- 发热
- 谵语
- 头痛
- 咳
- 下利
- 烦
- 悸

### 2. 脉象词

也非常重要，因为很多条文关系都靠脉象串起来。

示例：

- 脉浮
- 脉紧
- 脉弦细
- 脉沉
- 脉弱

说明：

- 如果原文是“脉浮紧”，可以直接写“脉浮紧”。
- 如果后续需要更细，也可以拆成“脉浮”“脉紧”，但先保证至少有一个准确词。

### 3. 证候 / 病机词

这类词能帮助关系图从“症状”走向“理解”。

示例：

- 少阳
- 阳明
- 胃不和
- 表证
- 里证
- 阴阳两虚

说明：

- 优先用原文里直接出现的词。
- 不要一开始就自己扩写成很长的解释句。

### 4. 方名词

如果条文里明确提到方子，可以加。

示例：

- 桂枝汤
- 小建中汤
- 黄芪建中汤

### 5. 部位词

当部位确实和理解相关时可以加。

示例：

- 胃
- 胸
- 腹
- 心下

## 三、不建议添加哪些关键词

下面这些词太泛，通常不要直接作为关键词。

- 病
- 证
- 热
- 寒
- 虚
- 实
- 人
- 者
- 时

原因：

- 这些词在太多文本里都会出现。
- 加进去以后，关系图会变得很乱。
- 右侧命中虽然可能变多，但价值不高。

项目里已经单独准备了一个黑名单文件：

- `data/keyword-blacklist.json`

以后如果发现新的“过泛词”，可以继续往这里加。

## 四、推荐写法

### 1. 尽量用原文里的词

推荐：

- 恶寒
- 胃不和
- 谵语

不推荐：

- 非常怕冷
- 胃气不调
- 神志不清胡言乱语

原因：

- 原文词更稳定。
- 后续做自动匹配更容易。

### 2. 长短要适中

太短不行，太长也不行。

推荐：

- 头痛
- 发热
- 脉弦细
- 胃不和

不推荐：

- 痛
- 热
- 少阳不可发汗发汗则谵语

### 3. 原则上不要自己造同义词

如果原文写“恶寒”，就优先写“恶寒”。

不要同时写：

- 恶寒
- 怕冷
- 畏寒

除非你非常确定它们都要作为不同入口保留。

## 五、一个条文加几个关键词合适

建议先加 3 到 8 个。

如果条文很短：

- 先加 3 到 5 个。

如果条文信息很多：

- 可以加到 6 到 8 个。

不建议一上来加十几个，因为：

- 很容易把泛词也塞进去。
- 后面排查问题会更难。

## 六、给一个完整示例

原文：

> 少阳不可发汗，发汗则谵语，此属胃，胃和则愈；胃不和，烦而悸。

推荐关键词：

- 少阳
- 谵语
- 胃
- 胃不和
- 烦
- 悸

说明：

- “少阳”是证候词。
- “谵语”“烦”“悸”是症状词。
- “胃”“胃不和”是病机或部位相关词。

## 七、添加关键词时的操作顺序

推荐统一按这个顺序来：

1. 先打开条文。
2. 先读原文，不要急着加词。
3. 圈出最重要的症状词。
4. 再找脉象词、证候词、方名词。
5. 检查有没有过泛词。
6. 保存后重新看右侧关联是否合理。

## 八、什么时候要去补关联解析

如果你已经给条文加了关键词，但右侧还是没有命中，通常说明不是条文错了，而是“关联解析那边还没有这个词”。

这时就去做两件事之一：

- 优先补对应的关联解析 JSON。
- 如果暂时没整理成 JSON，至少保证关联解析 TXT 里确实能搜到这个词。

## 九、给新手的最简单判断法

如果你拿不准一个词能不能加，就问自己 3 个问题：

1. 这个词是不是原文里真的有？
2. 这个词是不是有明确意思？
3. 这个词以后拿去搜别的经典或解析，值不值得？

如果 3 个问题都回答“是”，通常就可以加。

如果你还是不确定，优先保守一点，先不加泛词。
