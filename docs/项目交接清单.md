## 1. 这个项目是做什么的

这个项目是一个中医经典关系图项目。

它的核心逻辑是：

1. 经典条文里有关键词
2. 系统拿这些关键词去关联解析资料里查
3. 如果查到同名关键词，就建立关系
4. 最后把“条文 -> 关键词 -> 来源片段”显示在页面上

项目重点不是复杂知识图谱。

重点是：

- 条文
- 关键词
- 来源片段

平时优先维护：

[data](C:\Users\hxst01\Documents\aicoding\jingfangxingtu\data)

数据库文件在：

[app.db](C:\Users\hxst01\Documents\aicoding\jingfangxingtu\storage\app.db)

## 9. 接手时最容易踩的坑

### 坑 1：改了 `dist/` 里的文件

真正该改的是：

[data](C:\Users\hxst01\Documents\aicoding\jingfangxingtu\data)

不是：

[dist/data](C:\Users\hxst01\Documents\aicoding\jingfangxingtu\dist\data)

### 坑 2：以为数据库是唯一数据源

不是。

当前项目仍然是：

`JSON（数据文件格式）/ TXT（文本文件）为主，数据库为运行时副本。`

### 坑 3：只放了关联解析文件，但没登记配置

只把文件放进：

[data/关联解析](C:\Users\hxst01\Documents\aicoding\jingfangxingtu\data\关联解析)

还不够。

必须登记到：

[guanlianjiexiconfig.json](C:\Users\hxst01\Documents\aicoding\jingfangxingtu\data\guanlianjiexiconfig.json)

### 坑 4：关键词太泛

例如：

- 病
- 热
- 寒

这类词会让图太乱。

## 10. 常用命令

```bash
npm run dev
```

### 类型检查

```bash
npm run lint
```

### 构建

```bash
npm run build
```

### 初始化数据库

```bash
npm run db:init
```

### 导入经典

```bash
npm run db:import:shanghan
npm run db:import:jingui
npm run db:import:wenbing
```

### 导入关联解析

```bash
npm run db:import:relations
```

### 一次性重建数据库

```bash
npm run db:seed
```
