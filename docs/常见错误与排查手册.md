# 常见错误与排查手册

这份文档专门用来解决一个问题：

`项目出了问题时，先查什么，后查什么。`

这份手册尽量按“新手也能一步一步照着查”的方式写。

不要急着一下子怀疑代码坏了。

很多问题其实只是：

- 路径写错
- JSON 格式错
- 关键词没勾选
- 文件没登记
- 页面没刷新

---

## 1. 总原则

排查问题时，请按下面顺序来：

1. 先看文件在不在
2. 再看文件内容对不对
3. 再看配置有没有登记
4. 再看页面有没有刷新
5. 最后才看数据库

这很重要。

因为这个项目的数据源头主要还是 `data/`。

---

## 2. 页面打开后是空白，怎么办

这是最吓人的一种情况，但不一定最严重。

### 第一步：先确认开发服务有没有启动

运行：

```bash
npm run dev
```

然后打开：

[http://localhost:3000](http://localhost:3000)

### 第二步：检查最近是不是刚改过 JSON

如果刚改过：

- `jingdianconfig.json`
- `guanlianjiexiconfig.json`
- 某条条文 JSON

那很可能是 JSON 格式写错了。

### 第三步：运行类型检查和构建

```bash
npm run lint
npm run build
```

如果这两步报错，优先修报错。

### 第四步：检查是不是路径写错导致页面拿不到数据

重点看：

- `dataFile`
- 关联解析文件名
- 配置里的 `fileBaseName`

## 3. 新增了一本经典，但左侧没有显示

请按下面顺序排查。

### 第一步：看 `jingdianconfig.json` 里有没有这本书

打开：

[jingdianconfig.json](C:\Users\hxst01\Documents\aicoding\jingfangxingtu\data\jingdianconfig.json)

确认有没有类似：

```json
{
  "id": "piweilun",
  "name": "《脾胃论》",
  "chapters": []
}
```

如果根本没有，左侧当然不会显示。

### 第二步：确认 JSON 格式没错

最常见的是：

- 最后一个对象后面多了逗号
- 少了逗号
- 少了引号

### 第三步：确认 `chapters` 不是空的

有的情况下书会出现，但没有可点内容。

建议至少放一条条文。

### 第四步：刷新页面

有时候文件已经改对了，只是页面还是旧状态。

## 4. 左侧出现了条文，但点进去正文是空的

这种问题最常见。

### 第一步：检查 `dataFile`

例如配置里写：

```json
"dataFile": "/data/经典/伤寒论/265.json"
```

那真实文件就必须在：

[265.json](C:\Users\hxst01\Documents\aicoding\jingfangxingtu\data\经典\伤寒论\265.json)

### 第二步：检查文件名是不是一致

配置写的是：

```text
265.json
```

结果真实文件叫：

```text
256.json
```

这就会导致正文空白。

### 第三步：检查 JSON 文件里是不是缺字段

至少应该有：

- `id`
- `title`
- `content`
- `translation`
- `keywords`

## 5. 关键词已经加了，右侧却没有命中

这是最常见的问题之一。

请按下面顺序查。

### 第一步：确认这个关键词已经真的存在于条文里

直接看页面的关键词列表。

如果列表里都没有这个词，说明添加没成功。

### 第二步：确认关键词有没有勾选

如果这个词没勾选，它就不会参与当前匹配。

### 第三步：确认“来源片段”开关有没有打开

如果右侧来源片段被关闭，右侧看起来就像“没命中”。

### 第四步：去关联解析里查有没有同名词

例如关键词是：

```text
恶寒
```

那就看：

- JSON 里有没有 `恶寒`
- TXT 里有没有 `恶寒`

### 第五步：确认来源有没有登记

文件虽然放进了：

[data/关联解析](C:\Users\hxst01\Documents\aicoding\jingfangxingtu\data\关联解析)

但如果没有登记到：

[guanlianjiexiconfig.json](C:\Users\hxst01\Documents\aicoding\jingfangxingtu\data\guanlianjiexiconfig.json)

系统不会去查它。

### 第六步：确认关键词写法是不是一致

例如条文里是：

```text
恶寒
```

但解析里写的是：

```text
恶寒者
```

这不一定能自动对上。

---

## 6. 为什么有 JSON 也有 TXT，却还是没命中

现在规则是：

1. 某个关键词先查 JSON
2. 这个关键词在 JSON 没有，再查 TXT

所以要排查的是：

- 这个关键词是不是在 JSON 里没有
- TXT 里有没有这个词

不是简单地看“这个来源有没有 JSON”。

## 7. 关联解析文件放进去了，但系统没反应

### 第一步：确认文件名和配置一致

配置：

```json
{
  "fileBaseName": "437-黄帝内经素问"
}
```

文件就应该是：

- `437-黄帝内经素问.json`
- `437-黄帝内经素问.txt`

### 第二步：确认配置里已经登记

很多人把文件放进去就结束了。

这还不够。

必须登记到：

[guanlianjiexiconfig.json](C:\Users\hxst01\Documents\aicoding\jingfangxingtu\data\guanlianjiexiconfig.json)

### 第三步：确认文件编码和内容正常

如果文件内容本身就是乱码，命中和展示都会有问题。

## 8. 我改了 JSON，但页面没变化

### 第一步：先刷新页面

这是最简单也最容易忘的一步。

### 第二步：确认改的是 `data/` 里的文件，不是 `dist/` 里的文件

真正应该改的是：

[data](C:\Users\hxst01\Documents\aicoding\jingfangxingtu\data)

不是：

[dist/data](C:\Users\hxst01\Documents\aicoding\jingfangxingtu\dist\data)

### 第三步：确认是不是改错文件了

例如你以为自己改的是 `265.json`，结果改成了别的条文。

### 第四步：确认当前页面打开的就是这条条文

不要改的是 A 条文，却盯着 B 条文看结果。

---

## 9. 数据库导入后，数量不对怎么办

### 第一步：先确认 `data/` 文件本身对不对

数据库导入只是“搬运”。

如果源文件错了，导进去也会错。

### 第二步：确认导入命令有没有执行正确

例如：

```bash
npm run db:import:shanghan
npm run db:import:jingui
npm run db:import:wenbing
npm run db:import:relations
```

### 第三步：如果你刚做过很多改动，直接重建一次

```bash
npm run db:seed
```

这通常比猜哪一步漏了更快。

## 10. 添加关键词成功了，但删除后还是感觉没变化

请看下面几种可能。

### 可能 1：你删掉的是一个没有勾选的词

如果这个词本来就没参与当前筛选，那右侧变化可能不大。

### 可能 2：还有别的关键词也命中了同样来源

例如：

- `少阳` 命中了某条来源
- `谵语` 也命中了同一条来源

你删掉 `谵语` 后，这条来源仍可能因为 `少阳` 继续出现。

### 可能 3：你删的是当前条文上的关键词，不是全局关键词

别的条文里同名词仍然存在。

---

## 11. 什么时候需要怀疑数据库，什么时候不要怀疑

### 先不要急着怀疑数据库的情况

- 页面刚改完 JSON 没刷新
- 关键词没勾选
- 文件没登记
- 路径写错

这些更常见。

### 可以开始看数据库的情况

- 文件模式已经确认没问题
- 但 API 返回和页面表现明显不一致
- 导入后数量不对
- 某本书在文件里有，数据库里没有

---

## 12. 新手排查的万能顺序

如果你完全不知道从哪查，就按这个万能顺序来：

1. 看页面有没有刷新
2. 看你改的是不是 `data/` 里的正确文件
3. 看 JSON 格式有没有错
4. 看配置里有没有登记
5. 看路径和文件名是否一致
6. 看关键词是否勾选
7. 看关联解析里有没有同名词
8. 最后再跑：

```bash
npm run lint
npm run build
```

如果还不放心，再跑：

```bash
npm run db:seed
```

---

## 13. 最后记住一句话

排查时最稳的办法永远是：

`先查文件和配置，再查页面状态，最后才查数据库。`
