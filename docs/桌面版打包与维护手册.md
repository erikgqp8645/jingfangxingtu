# 桌面版打包与维护手册

这份手册专门说明 Windows 桌面版怎么打包、安装后数据放在哪里、以后怎么备份和迁移。

如果你只想记住最重要的一句话，请记住：

`不要直接改安装目录里的文件，桌面版真正会被修改的数据在用户目录里。`

---

## 1. 这份手册适合谁

这份手册适合下面几类人：

- 你想把项目打包成 Windows 安装包
- 你已经安装了桌面版，想知道数据放在哪
- 你后面还要继续维护关键词、经典数据、SQLite 数据库
- 你想给另一台电脑迁移同一份数据

---

## 2. 现在桌面版的工作方式

桌面版已经不是“单纯把网页包起来”。

现在的桌面版包含 3 部分：

- 前端界面
- 本地服务
- 本地可写数据

桌面版第一次启动时，会做一件很重要的事情：

1. 从安装包里读取内置的数据
2. 把这些数据复制到当前 Windows 用户自己的目录
3. 以后真正读写的都是这个用户目录里的数据

这样做的好处是：

- 安装目录可以保持干净
- 升级应用时不容易把用户自己维护的数据覆盖掉
- 后期备份更简单

---

## 3. 打包命令

### 3.1 生成图标资产

```powershell
npm run assets:desktop
```

这个命令会生成：

- `assets/desktop/icon.png`
- `assets/desktop/icon.ico`

源设计文件在：

- `assets/desktop/icon-source.svg`

如果以后你想换图标，先改这个 SVG，再执行上面的命令重新生成。

### 3.2 生成目录版

```powershell
npm run build:desktop:dir
```

这个命令会生成一个可直接运行的目录版。

输出位置：

- `release/win-unpacked/`

目录版主程序：

- `release/win-unpacked/JingfangRelations.exe`

### 3.3 生成安装包

```powershell
npm run build:desktop
```

输出位置：

- `release/经方关系图 Setup 0.1.0.exe`

这个文件就是可以分发给别人安装的 Windows 安装包。

## 4. 安装后，数据到底放在哪里

安装后真正会被修改的数据，不在安装目录里。

默认在 Windows 用户目录下，一般是：

```text
%APPDATA%\经方关系图\workspace\
```

通常展开后类似：

```text
C:\Users\你的用户名\AppData\Roaming\经方关系图\workspace\
```

这个目录里至少会有两类内容：

```text
workspace/
├─ data/
└─ storage/
   └─ app.db
```

含义如下：

- `data/`：可维护的经典 JSON、关联解析 JSON/TXT、配置文件
- `storage/app.db`：SQLite 数据库

---

## 5. 哪些地方能改，哪些地方不要改

### 5.1 可以维护的地方

后期维护主要看这两个位置：

- 用户目录里的 `data/`
- 用户目录里的 `storage/app.db`

### 5.2 不建议手工修改的地方

不要把安装目录当成维护入口。

例如不要优先改这些位置：

- `C:\Program Files\...`
- 安装包解压目录里的 `resources`
- `release/win-unpacked/` 里的资源文件

原因很简单：

- 升级时可能被覆盖
- 权限可能受限
- 容易和真实运行数据混淆

---

## 6. 如果我要备份，应该备份什么

最稳妥的备份方法是直接备份整个用户工作目录：

```text
%APPDATA%\经方关系图\workspace\
```

如果你想最小化备份，至少要备份：

- `data/`
- `storage/app.db`

建议做法：

1. 先完全退出桌面应用
2. 复制整个 `workspace/` 目录到别的盘
3. 备份文件名里写上日期

例如：

```text
workspace-backup-2026-04-22
```

## 7. 如果我要迁移到另一台电脑

最简单的迁移方法：

1. 在新电脑安装桌面版
2. 先启动一次，再退出
3. 找到新电脑的 `%APPDATA%\经方关系图\workspace\`
4. 用旧电脑备份出来的 `workspace/` 覆盖它
5. 再次启动应用

这样迁移后，一般会同时带过去：

- 经典数据
- 关键词维护结果
- SQLite 数据库内容

---

## 8. 如果我要升级桌面版

建议升级顺序：

1. 先备份 `%APPDATA%\经方关系图\workspace\`
2. 再安装新版本
3. 安装完成后检查关键词、条文、关系结果是否正常

因为现在桌面版真正的数据在用户目录里，所以正常升级不应该覆盖你维护过的数据。

但养成先备份再升级的习惯，最稳。

---

## 9. 常见问题

### 9.1 为什么我改了安装目录里的 JSON，没有效果

因为桌面版实际读写的是用户目录里的 `workspace/data/`，不是安装目录里的内置资源。

### 9.2 为什么我卸载再安装，之前的数据还在

因为数据在 `%APPDATA%` 用户目录，不在安装目录里。

### 9.3 如果我想彻底重置桌面版怎么办

先备份，然后删除：

```text
%APPDATA%\经方关系图\workspace\
```

下次启动时，应用会重新从安装包内置资源复制一份新的基础数据。

## 10. 现在这套桌面结构的优点

现在这套结构对后期维护有几个明显好处：

- 安装包和用户数据分开
- 数据备份路径明确
- 迁移到新电脑简单
- 后面继续加“导入”“修复”“校验”“备份”功能更自然
